# Telegram Tools Pack

Набор безопасных инструментов для работы с Telegram в твоём AI-офисе. Устанавливается одной фразой в Claude Code.

## Что умеет

### ✅ Парсинг публичных каналов (skill `telegram-channel-parser`)
Без API-ключей, без токенов, zero config. Парсит любой публичный канал через веб-превью `t.me/s/`.

**Что можно:**
- Читать посты из любого публичного канала
- Получать метрики (просмотры, реакции)
- Делать дайджесты по категориям (AI / конкуренты / новости — настраивается через `.env`)
- Сравнивать каналы, находить топ-посты, искать по ключевому слову
- Экспорт в CSV / JSON

**Типичные команды в Claude:**
- *«Дайджест AI-каналов за неделю»*
- *«Топ-10 постов из @linear.app за месяц»*
- *«Найди упоминания НейроШтаб в каналах X, Y, Z»*
- *«Сравни активность каналов A и B»*

### ✅ Парсинг HTML-экспорта (скрипт `parse-html-export`)
Если выгрузил историю СВОЕГО канала через Telegram Desktop → приложение создаёт HTML-файл → скрипт превращает его в структурированный JSON + Markdown (посты, реакции, медиа, ссылки).

**Когда нужно:** сделать контент-инвентаризацию старых постов своего канала для базы знаний / учить агентов твоему голосу.

## Установка за 30 секунд

Открой свой AI-офис в Claude Code и скажи:

> **установи телеграм-инструменты**

Или напиши развёрнутый промт:

```
Подключи мне Telegram Tools Pack из github.com/rusanovproject-dotcom/ai-office-packs.

Сделай так:
1. Скачай все файлы из _tool-packs/telegram/skills/telegram-channel-parser/
   и положи в ~/.claude/skills/telegram-channel-parser/
2. Скачай _tool-packs/telegram/extras/parse-html-export/parse_telegram.py
   в ~/workspace/tools/telegram/parse_telegram.py
3. Оповести меня живой фразой что готово.

Базовый URL: https://raw.githubusercontent.com/rusanovproject-dotcom/ai-office-packs/main/_tool-packs/telegram/
GitHub API: https://api.github.com/repos/rusanovproject-dotcom/ai-office-packs/contents/_tool-packs/telegram
```

После установки можно сразу: *«Дайджест канала @linear.app за неделю»*.

## Настройка дайджестов (опционально)

Хочешь чтобы команда «дайджест AI за неделю» работала без ввода каналов каждый раз?

1. Открой `~/.claude/skills/telegram-channel-parser/config/.env.example`
2. Скопируй в `.env` (в той же папке)
3. Пропиши свои категории:

```bash
TG_CATEGORIES=ai,competitors

TG_CHANNELS_AI_LABEL="AI и технологии"
TG_CHANNELS_AI=channel1,channel2,channel3

TG_CHANNELS_COMPETITORS_LABEL="Конкуренты"
TG_CHANNELS_COMPETITORS=competitor1,competitor2
```

Дальше говоришь «дайджест конкурентов» — Claude знает какие каналы.

## Что в паке НЕТ (и почему)

### ❌ Доступ к приватным каналам через MTProto (chigwell/telegram-mcp)

Существуют MCP-серверы которые дают полный доступ к твоему Telegram-аккаунту (чтение приватных каналов, отправка сообщений, редактирование профиля). Самый известный — `chigwell/telegram-mcp` (989 ⭐).

**Мы прогнали по нему полный security-аудит.** Код **чистый** (нет backdoors, нет telemetry, автор верифицирован), но **модель угроз высокая:**

- Session string = portable bearer token без IP-привязки. Утечка = полный захват аккаунта
- 100+ tools включая удаление сообщений и уход из групп — prompt injection через входящий чат может их дёрнуть
- PR #89 с защитой от prompt injection ещё не смержен

**Если очень хочется подключить MTProto** — сначала прочитай security-гайд:
`knowledge/research/2026-04-23_telegram-mcp-security-audit.md` в своём офисе.

Минимум что обязательно:
- Отдельный Telegram-аккаунт (не основной)
- Виртуальный номер (~1-3$ за SIM)
- Docker с `--cap-drop=ALL --read-only`
- Альтернатива: `sparfenyuk/mcp-telegram` (174 ⭐) — read-only по умолчанию, encrypted session

### ❌ Постинг через Bot API

Тоже требует секретов (bot token) и осознанной настройки. Пока не включаем в массовый пак.

## Частые вопросы

**Работает ли с Claude Free / без Claude Pro?**
Да. Скилл `telegram-channel-parser` работает через обычный curl + BeautifulSoup — никакой Claude API не нужен в скилле. Claude только запускает скрипты и парсит результат.

**Можно ли читать закрытые каналы?**
Нет, только публичные (у которых есть страница `t.me/s/{username}`). Для закрытых нужен MTProto — см. раздел «Что в паке нет».

**Нужно ли подключать бота / получать токены?**
Нет. Полностью zero config. Работает сразу после установки.

**Как часто обновляются данные?**
Каждый запрос — реальное время. Скилл не кэширует посты между запросами (только техническое кеширование на 1-2 минуты чтобы не колотить по серверу).

**Что если Telegram заблокирует доступ?**
Rate limit 1.5 секунды между запросами — Telegram не ругается. Если парсишь 100+ каналов за раз — пауза больше 3 минут на категорию, всё ок.
